File Format Requirements

  • File must be in CSV (.csv) or Excel (.xlsx, .xls) format
  • First row must contain column headers
  • Required columns: site_name, mobile, job_no, city
  • Optional columns: email, phone, site_address, office_address, same_as_site_address, contact_person_name, designation, pin_code, country, province_state, routes, branch, sector, handover_date (or handover_date_str), billing_name, latitude, longitude, notes, generate_license_now
  • Foreign key fields (province_state, city, routes, branch) must match existing records by their value/name
  • Validation Rules (same as Add Customer form):
  • • Site Name: Required
  • • Mobile: Required, must be exactly 10 digits, must be unique
  • • Job No: Required, must be unique
  • • City: Required, must match an existing city name
  • • Email: Optional, must be unique if provided
  • • Phone: Optional, must be exactly 10 digits if provided, must be unique if provided
  • • Province/State: Optional, must match an existing province/state name
  • • Routes: Optional, must match an existing route name
  • • Branch: Optional, must match an existing branch name
  • • Sector: Optional, must be either "government" or "private"
  • • Handover Date: Optional, format YYYY-MM-DD (same as export format)
  • • Latitude: Optional, must be between -90 and 90 degrees
  • • Longitude: Optional, must be between -180 and 180 degrees
  • • Same as Site Address: Optional boolean (true/false, yes/no, on/off, 1/0). If true, office_address will be set to site_address
  • • Generate License Now: Optional boolean (true/false, yes/no, on/off, 1/0)
  • • Note: File uploads (uploads_files) cannot be handled in bulk import. Upload files individually after import.
